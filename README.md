## 📖 About

**Join.ai** is a full-stack AI meeting platform that lets users create personalized AI agents, schedule video meetings with them, and receive intelligent post-meeting summaries. Think of it as your own AI-powered meeting companion — you define the agent's personality and instructions, hop into a real-time video call, and when you're done the platform automatically transcribes the session, identifies speakers, and generates a structured summary using AI.

Built with a modern TypeScript-first stack, the app features real-time video via **Stream Video SDK**, AI-powered voice responses via **OpenAI's Realtime API**, background job processing with **Inngest**, and a serverless PostgreSQL database on **Neon**.

### Step-by-Step Flow

1. **🔐 Authentication** — Users sign in via GitHub or Google OAuth. Sessions are managed by Better Auth with Drizzle-backed storage.

2. **🤖 Agent Creation** — Users create custom AI agents by defining a name and detailed instructions that shape the agent's personality and behavior.

3. **📅 Meeting Setup** — Users schedule meetings by selecting an agent and setting meeting details. A unique meeting ID is generated using nanoid.

4. **📹 Joining a Call** — Users enter a lobby with camera/mic preview, then join. Stream Video SDK sets up the WebRTC call. A webhook (`call.session_started`) triggers the backend to connect the OpenAI Realtime API, loading the agent's custom instructions.

5. **🗣️ Real-Time Conversation** — The AI agent responds with live voice using OpenAI's Realtime API. Users have a natural, real-time conversation with their AI agent.

6. **📞 Call End & Processing** — When the call ends, Stream fires webhook events. The app transitions the meeting to `processing` status, fetches the transcript, identifies speakers, and uses Inngest to run a background summarization job.

7. **📊 Summary Generation** — The Inngest function fetches the transcript, maps speaker IDs to actual names, then uses an AI summarizer agent (GPT-3.5-Turbo) to produce a structured summary with Overview and Notes sections.

8. **📋 Dashboard View** — Users view completed meetings with transcripts, recordings, and AI-generated summaries—all from the dashboard.

## 🧠 What I Learned

Building Join.ai was a deep dive into modern full-stack development. Here are the key takeaways:

### Real-Time Architecture
- Learned how **WebRTC** works under the hood via Stream Video SDK — handling media streams, peer connections, and call state management.
- Integrated **OpenAI's Realtime API** for live voice AI — understanding WebSocket connections, session management, and real-time audio streaming.

### Event-Driven Backend
- Built an **event-driven architecture** using Stream webhooks + Inngest, learning how to reliably process asynchronous events like transcription, recording, and session lifecycle.
- Understood the importance of **idempotent event handlers** and proper webhook signature verification for security.

### Type-Safe Full Stack
- Achieved **end-to-end type safety** using tRPC + Drizzle + Zod — from database schema to API layer to frontend forms, every layer shares types.
- Learned how `inferRouterOutputs` in tRPC eliminates manual type definitions across the stack.

### Authentication & Payments
- Implemented **multi-provider OAuth** with Better Auth, handling session management, token refresh, and secure cookie handling.
- Integrated **Polar.sh** for premium subscriptions, learning about payment webhooks and entitlement-based feature gating.

### Background Processing
- Used **Inngest** for reliable background job processing — step functions, retries, and event orchestration for the transcript → summary pipeline.
- Built an **AI agent pipeline** using Inngest Agent Kit to chain transcript fetching, speaker identification, and AI summarization into a robust workflow.

## 🚀 Getting Started

### Prerequisites

- **Node.js** ≥ 20
- **npm** (comes with Node.js)
- A **Neon** PostgreSQL database
- **Stream** Video API keys
- **OpenAI** API key
- **GitHub** & **Google** OAuth credentials
- **Polar.sh** account (for premium features)
- **ngrok** (for local webhook development)

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/join-ai.git
cd join-ai
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Set Up Environment Variables

Create a `.env` file in the root directory:

```env
# Database
DATABASE_URL=your_neon_postgresql_connection_string

# Auth
BETTER_AUTH_SECRET=your_random_secret_key
BETTER_AUTH_URL=http://localhost:3000

# OAuth - GitHub
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret

# OAuth - Google
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# App
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Stream Video
NEXT_PUBLIC_STREAM_VIDEO_API_KEY=your_stream_api_key
STREAM_VIDEO_SECRET_KEY=your_stream_secret_key

# OpenAI
OPENAI_API_KEY=your_openai_api_key

# Polar (Premium)
POLAR_ACCESS_TOKEN=your_polar_access_token
```

### 4. Push Database Schema

```bash
npm run db:push
```

### 5. Start the Development Server

```bash
npm run dev
```

The app will be running at **[http://localhost:3000](http://localhost:3000)**.

### 6. Set Up Webhooks (Local Development)

In a separate terminal, start ngrok to receive Stream webhook events:

```bash
npm run dev:webhook
```

Configure the ngrok URL as your Stream webhook endpoint in your Stream Dashboard.

### 7. (Optional) Run with Docker

```bash
docker compose up --build
```
